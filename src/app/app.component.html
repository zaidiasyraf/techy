@ConfigurationProperties

Dealing with environment variable is not new things for spring developer. Often we use @value to inject data from config file into the field

However there is issue if we need to have same configuration data at multiple place. Following above way, we just need to use @value at all of those places.

But then there is better way to inject the data. Fret not, we have @ConfigurationProperties This resolve the weakness of @value.

Futhermore all related config can be consolidated into one class and can be injected to any intended component. Have I also mentioned that this anotation also make it easier to unit test too?
